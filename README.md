## Arbetsflöde med GitHub Issues & Brancher

1. Skapa en issue för din uppgift och assigna dig själv.
2. Klicka på "Create a branch" i issuen för att skapa en branch kopplad till din uppgift.
3. Byt till din nya branch lokalt med (står alltid hur man gör i instruktionen du får):

   ```bash
   git fetch
   git checkout <branch-namn>
   ```

   _Byt ut `<branch-namn>` till namnet på din branch._

4. **Glöm inte att köra `git pull` regelbundet för att hålla din branch uppdaterad!**

---

## Viktiga regler

- **Pusha aldrig direkt till `main`.**
- **Gör alltid en pull request från din branch till `develop` när du är klar med din uppgift.**
- **Se till att din kod är uppdaterad med senaste ändringarna från `develop` innan du gör en pull request.**
